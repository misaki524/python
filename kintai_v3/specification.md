### 4. 機能詳細仕様

#### 4.1 F-001: 出勤打刻（1日複数回対応）

| 項目 | 内容 |
|------|------|
| **トリガー** | 「▶ 出勤」ボタン押下 |
| **処理関数** | `clockIn()` |
| **事前条件** | 当日の未退勤セッション（退勤時刻が空の行）が存在しないこと |
| **処理内容** | ① 当日の全出勤記録を検索 → ② 未退勤セッションがないことを確認 → ③ Attendanceシートに新規行を追加 |
| **記録内容** | 日付, 出勤時刻, ステータス=「出勤中」 |
| **成功時レスポンス** | `{ success: true, message: "出勤しました (HH:mm) [第N回]" }` |
| **失敗時レスポンス** | `{ success: false, message: "既に出勤中です。先に退勤してください" }` |
| **UI反映** | ステータスバッジ → 「勤務中」（緑）、出勤ボタン無効化 |
| **備考** | 退勤済の状態から再度出勤可能（1日に複数セッションを記録できる） |

#### 4.2 F-002: 退勤打刻（最新セッションを閉じる）

| 項目 | 内容 |
|------|------|
| **トリガー** | 「⏹ 退勤」ボタン押下 → **カスタムモーダル**（「退勤しますか？」+ 👋アイコン）→ 「退勤する」ボタン押下 |
| **処理関数** | `clockOut()` |
| **事前条件** | ① 当日の未退勤セッションが存在すること → ② 休憩中でないこと |
| **処理内容** | ① 当日の全出勤記録から最新の未退勤セッションを検索 → ② 退勤時刻を記録 → ③ そのセッション中の休憩時間合計を取得 → ④ 勤務時間を計算 → ⑤ ステータスを「退勤済」に更新 |
| **勤務時間計算式** | `勤務時間 = 退勤時刻 - 出勤時刻 - 該当セッション中の休憩時間合計` |
| **成功時レスポンス** | `{ success: true, message: "退勤しました (HH:mm)\n勤務時間: H:mm" }` |
| **失敗時レスポンス** | `{ success: false, message: "出勤中の記録がありません" }` |
| **UI反映** | ステータスバッジ → 「退勤済」（グレー）、出勤ボタンのみ有効化（再出勤可能） |
| **確認UI** | ブラウザ標準の `confirm()` ではなく、カスタムモーダル（アニメーション付き）を使用 |

#### 4.3 F-003: 休憩開始

| 項目 | 内容 |
|------|------|
| **トリガー** | 「☕ 休憩開始」ボタン押下 |
| **処理関数** | `breakStart()` |
| **事前条件** | ① 未退勤セッションが存在すること（出勤中であること） → ② 現在休憩中でないこと |
| **処理内容** | ① 当日の全出勤記録から未退勤セッションの存在を確認 → ② 当日の未終了休憩記録を検索 → ③ 未記録の場合、Breaksシートに新規行を追加 |
| **記録内容** | 日付, 休憩開始時刻 |
| **成功時レスポンス** | `{ success: true, message: "休憩開始 (HH:mm)" }` |
| **失敗時レスポンス** | `{ success: false, message: "既に休憩中です" }` |
| **UI反映** | ステータスバッジ → 「休憩中」（オレンジ）、退勤ボタン無効化 |

#### 4.4 F-004: 休憩終了

| 項目 | 内容 |
|------|------|
| **トリガー** | 「💪 休憩終了」ボタン押下 |
| **処理関数** | `breakEnd()` |
| **事前条件** | 当日の未終了休憩記録が存在すること |
| **処理内容** | ① 当日の未終了休憩記録を検索（最新から） → ② 休憩終了時刻を記録 → ③ 休憩時間を計算 |
| **休憩時間計算式** | `休憩時間 = 休憩終了時刻 - 休憩開始時刻` |
| **成功時レスポンス** | `{ success: true, message: "休憩終了 (HH:mm)\n休憩時間: H:mm" }` |
| **失敗時レスポンス** | `{ success: false, message: "休憩中の記録がありません" }` |
| **UI反映** | ステータスバッジ → 「勤務中」（緑）に復帰 |
| **備考** | 1日に複数回の休憩が可能 |

#### 4.5 F-005: 日報作成・更新

| 項目 | 内容 |
|------|------|
| **トリガー** | 「💾 日報を保存」ボタン押下 |
| **処理関数** | `saveReport(reportText)` |
| **事前条件** | テキストエリアが空でないこと（フロント側バリデーション） |
| **処理内容** | ① 当日の既存日報を検索 → ② 存在する場合は更新 → ③ 存在しない場合は新規作成 |
| **入力形式** | 箇条書き（例：「・会議参加」「・資料作成」） |
| **成功時レスポンス（新規）** | `{ success: true, message: "日報を保存しました" }` |
| **成功時レスポンス（更新）** | `{ success: true, message: "日報を更新しました" }` |
| **備考** | 同日中は何度でも上書き更新可能 |

#### 4.6 F-006: ステータス表示

| ステータス | 条件 | バッジ色 | CSSクラス |
|-----------|------|---------|-----------|
| 未出勤 | 当日の出勤記録なし | 半透明白 | `status-none` |
| 勤務中 | 未退勤セッションあり＆休憩中でない | 緑 `#4CAF50` | `status-working` |
| 休憩中 | 未退勤セッションあり＆未終了の休憩記録あり | オレンジ `#FF9800` | `status-break` |
| 退勤済 | 全セッションが退勤済み | グレー `#9E9E9E` | `status-done` |

**ステータス判定ロジック（`currentStatus`）：**
- 出勤記録なし → 空文字（未出勤）
- 未退勤セッションあり＋未終了休憩あり → `'休憩中'`
- 未退勤セッションあり → `'出勤中'`
- 全セッション退勤済み → `'退勤済'`

#### 4.7 F-007: 本日情報取得

| 項目 | 内容 |
|------|------|
| **処理関数** | `getTodayStatus()` |
| **呼び出しタイミング** | ① 初回ページ読み込み時 → ② 各操作（出勤/退勤/休憩開始・終了）成功後 |
| **レスポンス構造** | 下記参照 |

```json name=response_schema.json
{
  "date": "yyyy-MM-dd",
  "currentTime": "HH:mm",
  "sessions": [
    {
      "num": 1,
      "clockIn": "HH:mm",
      "clockOut": "HH:mm | 空文字",
      "workHours": "H:mm | 空文字",
      "active": true | false
    }
  ],
  "totalWorkHours": "H:mm | 空文字",
  "currentStatus": "出勤中 | 休憩中 | 退勤済 | 空文字",
  "breaks": [
    {
      "start": "HH:mm",
      "end": "HH:mm | 空文字",
      "duration": "H:mm | 空文字"
    }
  ],
  "report": "日報テキスト | 空文字"
}
```

**v1.0.0 → v1.1.0 の変更点:**
- `attendance` オブジェクト → `sessions` 配列に変更（1日複数セッション対応）
- `totalWorkHours` フィールド追加（全セッション合計勤務時間）
- `currentStatus` フィールド追加（ステータスをサーバー側で判定）
- 各セッションに `num`（回数）と `active`（記録中かどうか）を追加
```

### 5. データベース設計（Google Sheets）

#### 5.1 Attendance シート（勤怠記録）

> **v1.1.0:** 1日に複数行（複数セッション）が存在可能。同じ日付で複数の出勤/退勤ペアが記録される。

| 列 | カラム名 | データ型 | 説明 | 例 |
|----|---------|---------|------|-----|
| A | 日付 | Date | 記録日（Date型で保存） | 2026-03-13 |
| B | 出勤時刻 | String/Date | HH:mm形式（Sheetsが自動変換する場合あり） | 09:00 |
| C | 退勤時刻 | String/Date | HH:mm形式（退勤前は空） | 12:00 |
| D | 勤務時間 | String | H:mm形式（退勤前は空・セッション単位） | 3:00 |
| E | ステータス | String | 「出勤中」または「退勤済」 | 出勤中 |

**データ例（1日2セッション）：**

| A | B | C | D | E |
|---|---|---|---|---|
| 2026-03-14 | 09:00 | 12:00 | 3:00 | 退勤済 |
| 2026-03-14 | 13:00 | 18:00 | 5:00 | 退勤済 |

#### 5.2 Breaks シート（休憩記録）

| 列 | カラム名 | データ型 | 説明 | 例 |
|----|---------|---------|------|-----|
| A | 日付 | Date | 記録日（Date型で保存） | 2026-03-13 |
| B | 休憩開始 | String | HH:mm形式 | 12:00 |
| C | 休憩終了 | String | HH:mm形式（休憩中は空） | 13:00 |
| D | 休憩時間 | String | H:mm形式（休憩中は空） | 1:00 |

#### 5.3 DailyReport シート（日報）

| 列 | カラム名 | データ型 | 説明 | 例 |
|----|---------|---------|------|-----|
| A | 日付 | Date | 記録日（Date型で保存） | 2026-03-13 |
| B | 内容 | String | 箇条書きテキスト | ・会議参加\n・資料作成 |

### 6. ボタン制御マトリクス

各状態におけるボタンの活性/非活性を定義する。

| 状態 | 出勤ボタン | 退勤ボタン | 休憩開始ボタン | 休憩終了ボタン | 日報保存ボタン |
|------|:--------:|:--------:|:----------:|:----------:|:----------:|
| 未出勤 | ✅ 有効 | ❌ 無効 | ❌ 無効 | ❌ 無効 | ✅ 有効 |
| 勤務中 | ❌ 無効 | ✅ 有効 | ✅ 有効 | ❌ 無効 | ✅ 有効 |
| 休憩中 | ❌ 無効 | ❌ 無効 | ❌ 無効 | ✅ 有効 | ✅ 有効 |
| 退勤済 | ✅ 有効 | ❌ 無効 | ❌ 無効 | ❌ 無効 | ✅ 有効 |

> **v1.1.0 変更:** 退勤済状態で出勤ボタンが有効になり、1日に複数回の出退勤が可能になりました。

### 8. バリデーション・エラーハンドリング

#### 8.1 フロントエンド

| チェック項目 | 処理 |
|------------|------|
| 日報が空 | トースト「日報を入力してください」を表示、サーバー呼び出しなし |
| 退勤ボタン押下 | カスタムモーダル（アニメーション付き）で確認、キャンセル時は処理中止 |
| API呼び出し中 | 押下したボタンを即時 `disabled` にし二重送信防止 |
| API失敗時 | トースト「エラーが発生しました」を表示、`resetButtonsToDefault()` でボタン復元 |

#### 8.2 バックエンド

| チェック項目 | 処理 |
|------------|------|
| 未退勤で出勤 | `{ success: false, message: "既に出勤中です。先に退勤してください" }` を返却 |
| 未出勤で退勤 | `{ success: false, message: "出勤中の記録がありません" }` を返却 |
| 休憩中に退勤 | `{ success: false, message: "休憩中は退勤できません。先に休憩を終了してください" }` を返却 |
| 未出勤で休憩開始 | `{ success: false, message: "出勤中ではありません" }` を返却 |
| 重複休憩開始 | `{ success: false, message: "既に休憩中です" }` を返却 |
| 未休憩で休憩終了 | `{ success: false, message: "休憩中の記録がありません" }` を返却 |

### 9. 非機能要件

| カテゴリ | 要件 |
|---------|------|
| **可用性** | Googleサービスの稼働に依存（SLA 99.9%） |
| **性能** | Apps Scriptの実行時間制限：6分/回（通常1-3秒で完了） |
| **セキュリティ** | デプロイ時「自分のみ」に制限、Googleアカウント認証 |
| **データ保全** | Google Sheetsの自動保存・バージョン管理を利用 |
| **対応デバイス** | iOS Safari / Android Chrome（モバイルファースト設計） |
| **タイムゾーン** | `Asia/Tokyo`（JST）固定 |
| **同時アクセス** | 1名のみ（考慮不要） |
| **データ保持期間** | 無期限（Google Sheets容量制限：1,000万セル） |

### 10. API一覧

| No. | 関数名 | 引数 | 戻り値型 | 説明 |
|-----|--------|------|---------|------|
| 1 | `doGet()` | なし | HtmlOutput | Webアプリのエントリーポイント |
| 2 | `clockIn()` | なし | Object | 出勤打刻（1日複数回可） |
| 3 | `clockOut()` | なし | Object | 退勤打刻（最新の未退勤セッションを閉じる） |
| 4 | `breakStart()` | なし | Object | 休憩開始 |
| 5 | `breakEnd()` | なし | Object | 休憩終了 |
| 6 | `saveReport(reportText)` | String | Object | 日報保存 |
| 7 | `getTodayStatus()` | なし | Object | 本日ステータス取得（sessions配列形式） |

**内部ヘルパー関数：**

| 関数名 | 説明 |
|--------|------|
| `toTimeStr_(cellValue)` | セル値をHH:mm文字列に安全変換（Date型/String型両対応） |
| `findTodayRows_(sheetName)` | 指定シートから当日行を検索（明示的列範囲で読み取り） |
| `calcDuration_(startTimeStr, endTimeStr)` | 時間差をH:mm形式で計算 |
| `durationToMinutes_(durationStr)` | H:mm文字列を分数に変換 |
| `minutesToDuration_(minutes)` | 分数をH:mm文字列に変換 |

**共通レスポンス形式（`getTodayStatus` 以外）：**

```json name=common_response.json
{
  "success": true | false,
  "message": "ユーザー向けメッセージ"
}

### 15. 技術的注意事項

#### 15.1 Google Sheetsのセル値変換問題

Google Sheetsは `"09:00"` のような文字列をDate型に自動変換する場合がある。これにより、セル値を文字列として扱うとNaN表示が発生する。

**対策:** `toTimeStr_(cellValue)` ヘルパー関数を全てのセル読み取り箇所で使用し、Date型/String型のどちらでも安全にHH:mm形式の文字列に変換する。

#### 15.2 getDataRange()の列範囲問題

`sheet.getDataRange()` は空列を含まないため、E列（ステータス）が空の場合に読み取れない。

**対策:** `findTodayRows_()` で `Math.max(sheet.getLastColumn(), 5)` により明示的に5列以上を読み取る。

#### 15.3 書き込み後のフラッシュ

`SpreadsheetApp.flush()` を全write操作後に呼び出し、書き込みの即時反映を保証する。
