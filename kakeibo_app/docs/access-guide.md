# アプリへのアクセス手順（ブラウザ／スマホ）

家計簿アプリは **ブラウザで動く Web アプリ** です。インストール不要で、PC でもスマホでも同じURLから使えます。
データは Google スプレッドシートに保存されるので、複数端末で開いても同じ内容が表示されます（※各端末で初回に設定画面のGAS URL登録が必要）。

アクセス方法は大きく2通りです。

| 方法 | こんな人向け | 難易度 |
|------|------------|--------|
| ① ネット公開（Netlify など） | スマホで毎日いつでも使いたい（**おすすめ**） | ★★☆ 初回のみ |
| ② 同じWi-Fiでローカルアクセス | 自宅のPCを起動しているときだけ使えればいい | ★☆☆ |

---

## ① ネットに公開してスマホで開く（おすすめ）

一度公開すれば、PCを起動していなくてもスマホからいつでもアクセスできます。
ここでは無料で一番かんたんな **Netlify Drop** を使う方法を説明します。

### 手順

1. **アプリをビルドする**（PCのターミナルで実行）

   ```bash
   cd kakeibo_app
   npm install      # 初回のみ
   npm run build
   ```

   → `kakeibo_app/dist/` フォルダが作られます。これが公開用のファイルです。

2. **Netlify Drop を開く**
   PCのブラウザで <https://app.netlify.com/drop> を開きます。
   （無料アカウントの作成を求められたら、Googleアカウントなどで登録）

3. **`dist` フォルダをドラッグ＆ドロップ**
   手順1で作った `dist` フォルダを、Netlify Drop のページにそのまま放り込みます。

4. **URLが発行される**
   `https://xxxxxx.netlify.app` のようなURLが表示されます。これがアプリのアドレスです。
   （Netlifyの管理画面から好きな名前に変更も可能）

5. **SPA用の設定（重要・1回だけ）**
   このアプリはページ遷移を内部で処理するため、リロード時に「Page not found」が出ないよう
   `kakeibo_app/public/_redirects` というファイルを作り、次の1行を書いておきます。

   ```
   /*    /index.html   200
   ```

   このファイルを置いてから `npm run build` し直すと、`dist` に自動で含まれます。
   （※この設定をしないと、スマホで再読み込みしたときにエラー画面になることがあります）

6. **更新したいとき**
   コードを直したら再度 `npm run build` → Netlifyの同じサイトに新しい `dist` をドロップすれば差し替わります。

> 💡 Netlify のほかに **Vercel** や **Cloudflare Pages** でも同様に公開できます。いずれも無料枠で十分です。

## ② 同じWi-Fiでローカルアクセス（PC起動中のみ）

PCで開発サーバーを立ち上げ、同じWi-Fiにつないだスマホからアクセスする方法です。
公開作業は不要ですが、**PCを起動してサーバーを動かしている間だけ**使えます。

### 手順

1. **PCで開発サーバーを起動**（`--host` を付けるのがポイント）

   ```bash
   cd kakeibo_app
   npm install          # 初回のみ
   npm run dev -- --host
   ```

   起動すると、こんな表示が出ます。

   ```
   ➜  Local:   http://localhost:5173/
   ➜  Network: http://192.168.x.x:5173/   ← これをスマホで使う
   ```

2. **PCのIPアドレスを確認**（Networkの行が出ない場合）

   Macのターミナルで：
   ```bash
   ipconfig getifaddr en0
   ```
   → 例 `192.168.1.23` のような番号が出ます。

3. **スマホを同じWi-Fiに接続**
   PCとスマホが**同じWi-Fiネットワーク**につながっていることを確認します。

4. **スマホのブラウザでアクセス**
   Safari や Chrome のアドレス欄に、手順1のNetworkのURL（例 `http://192.168.1.23:5173/`）を入力して開きます。

> ⚠️ うまく開けないとき
> - PCとスマホが同じWi-Fiか確認（ゲストWi-Fiは不可のことが多い）
> - PCのファイアウォールがポート `5173` をブロックしていないか確認
> - ビルド済みファイルで試す場合は `npm run build` → `npm run preview -- --host` でもOK

## 📱 スマホでアプリのように使う（ホーム画面に追加）

ブラウザで開いたあと「ホーム画面に追加」すると、アイコンから一発で開けてアプリのように使えます。

### iPhone（Safari）
1. アプリのURLを Safari で開く
2. 下部の **共有ボタン**（□から↑が出ているマーク）をタップ
3. **「ホーム画面に追加」** を選ぶ
4. 名前を確認して **「追加」**
5. → ホーム画面にアイコンが追加され、次回からはアイコンをタップするだけ

### Android（Chrome）
1. アプリのURLを Chrome で開く
2. 右上の **︙（メニュー）** をタップ
3. **「ホーム画面に追加」** を選ぶ
4. **「追加」** をタップ
5. → ホーム画面にアイコンが追加されます

## 初回に必要な設定（どの端末でも共通）

アプリを開いただけではデータは表示されません。**設定画面でGASのURLを登録**する必要があります。

1. アプリ下部メニューの **「設定」** を開く
2. **GAS WebアプリURL** を入力して「保存」
3. **「接続テスト＆シート自動作成」** ボタンを押す

> GAS URL の取得・デプロイ方法は、設定画面の「セットアップ手順」または
> [`gas-code.js`](gas-code.js) 冒頭のコメントを参照してください。
> この登録は端末（ブラウザ）ごとに必要です（URLは各端末のブラウザに保存されます）。

## よくある質問

**Q. PCを消すとスマホで開けなくなる？**
A. 方法②（ローカル）の場合はそうです。常に使いたいなら方法①（ネット公開）を選んでください。公開後はPCが消えていても開けます。

**Q. データはどこに保存される？**
A. Google スプレッドシートです。アプリ自体はデータを持たないため、端末を変えても同じデータが見られます（各端末でGAS URL登録は必要）。

**Q. 削除したカテゴリ／バイト先のデータは消える？**
A. 消えません。過去データはスプレッドシートに残り、「削除された項目です」と表示されます。
